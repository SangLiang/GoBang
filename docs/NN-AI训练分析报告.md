# NN AI 训练分析报告

## 概述

本文档记录了对五子棋神经网络AI（NN AI）的训练、测试和问题分析过程。通过对比自对弈训练与向Rule AI学习两种方式，发现了NN AI存在"钻空子"而非真正学会五子棋的问题。

## 三、问题分析

### 3.1 核心问题：NN AI "钻空子"

NN AI对Rule AI 100%胜率，但对人类很弱，说明NN AI学会了**针对Rule AI弱点的策略**，而非真正的五子棋技能。

### 3.2 棋型理解测试

#### 测试1：关键棋型识别（自对弈训练后）

| 棋型 | Rule AI评分 | NN AI权重 | 结论 |
|------|------------|----------|------|
| 活四 OOOO_ | 120,000 | 64.8 | ❌ 无法识别 |
| 冲四 OOOOX | 15,000 | 64.8 | ❌ 无法识别 |
| 活三 _OOO_ | 2,800 | 64.9 | ❌ 无法识别 |
| 随机空位 | ~35 | 133.6 | ❌ 反而更高 |

**结论**：自对弈训练的NN AI完全不理解五子棋的关键棋形。

#### 测试2：关键棋型识别（向Rule AI学习后）

| 棋型 | Rule AI评分 | NN AI权重 | 结论 |
|------|------------|----------|------|
| 活四 OOOO_ | 1,000,105 | 922.1 | ✅ 能识别 |

**结论**：向Rule AI学习后，NN AI学会了识别关键棋形。

### 3.3 防守与进攻能力测试

| 场景 | NN AI选择 | Rule AI选择 | 是否正确 |
|------|----------|------------|----------|
| 对手活三 | (7,10) | (7,6) | ✅ 正确 |
| 对手冲四 | (7,6) | (7,6) | ✅ 正确 |
| **自己活三进攻** | **(6,7)** | (7,6) | ❌ **错误** |
| 复杂局面 | (10,13) | (10,9) | 不同 |

**结论**：
- 防守能力：合格，能识别必须防守的位置
- 进攻能力：**有问题**，不能正确选择进攻点

## 四、问题根因分析

### 4.1 过拟合到单一对手

NN AI在与Rule AI的对弈中，可能发现了Rule AI的某种系统性弱点：

```javascript
// 可能的"钻空子"策略
// NN AI学会了某种特定走法，恰好能击败Rule AI
// 但这种走法对人类或其他对手无效
```

### 4.2 训练数据偏差

- 只与Rule AI对弈，缺乏多样性
- Rule AI的决策模式固定，NN AI可能学会了"套路"
- 人类玩家的走法更加灵活多变

### 4.3 网络结构局限

- 当前网络：22→32→1（相对简单）
- 特征维度：22维局部特征
- 可能缺乏全局视野和长期规划能力

## 八、灾难性遗忘问题（2026-04-03 更新）

### 8.1 问题描述

在交替训练过程中发现：**NN AI 学会新策略后会遗忘旧策略**。

**现象**：
```
第1轮：白棋 vs 黑方A → 白棋学会对策A
第2轮：黑棋 vs 白方B → 黑棋学会对策B（但忘了怎么应对A）
第3轮：白棋 vs 黑方C → 白棋又忘了之前的策略
```

### 8.2 根因分析

| 原因 | 说明 |
|------|------|
| **固定对手** | 每轮只面对一个固定策略，缺乏多样性 |
| **无历史回放** | 训练完就丢弃之前的经验 |
| **权重覆盖** | 新策略直接覆盖旧权重，没有保护机制 |

这是神经网络训练中的经典问题：**灾难性遗忘（Catastrophic Forgetting）**。

### 8.3 解决方案：多对手训练（推荐）

让进化方同时面对**多个历史版本**的对手：

```javascript
// 训练白方时，对手池包含：
var opponentPool = [
    opponent-black-v1.json,  // 初始版本
    opponent-black-v2.json,  // 第1轮训练后
    opponent-black-v3.json   // 第2轮训练后
];

// 每局随机选一个对手
var opponent = opponentPool[Math.floor(Math.random() * opponentPool.length)];
```

**优点**：
- AI 必须学会应对多种策略，不会遗忘
- 实现简单，不需要修改训练逻辑
- 自然形成"军备竞赛"，双方持续进化

**实现步骤**：
1. 保存历史对手版本（不覆盖，保存为新文件）
2. 训练时随机从对手池中选择
3. 保留最近 N 个版本，淘汰太旧的

### 8.4 其他方案（备选）

| 方案 | 优点 | 缺点 |
|------|------|------|
| **经验回放** | 保留历史经验 | 实现复杂，需修改训练逻辑 |
| **弹性权重巩固（EWC）** | 理论上最优 | 计算量大，实现复杂 |

## 九、结论

### 9.1 主要发现

1. **自对弈训练无效**：NN AI从0开始自对弈，无法学会五子棋基本棋形
2. **向Rule AI学习有效**：继续训练后NN AI能100%击败Rule AI
3. **存在"钻空子"问题**：NN AI学会了针对Rule AI弱点的策略，而非通用五子棋策略
4. **对人类表现差**：证实了NN AI并未真正学会五子棋
5. **灾难性遗忘**：交替训练时AI会遗忘旧策略，需多对手训练解决

### 9.2 核心问题

> **NN AI 通过学习Rule AI的弱点获得了高胜率，但这种"投机取巧"的策略对人类玩家无效。**

### 9.3 下一步行动

1. 实现多对手训练策略（解决灾难性遗忘）
2. 增加网络容量和训练代数
3. 引入更多样化的对手
4. 添加全局特征提升视野
